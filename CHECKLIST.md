## Что нужно сделать вам

### 1. Закоммитьте изменения
```bash
git add .
git commit -m "Fix Railway deployment - separate gunicorn from Flask"
git push origin main
```

### 2. Настройте Railway

#### Добавьте базу данных:
- [ ] Зайдите в проект на Railway
- [ ] Нажмите "New" → "Database" → "PostgreSQL"
- [ ] Дождитесь создания БД

#### Установите переменные окружения:
- [ ] Перейдите во вкладку "Variables"
- [ ] Добавьте `SECRET_KEY` (сгенерируйте надёжный ключ)
- [ ] Проверьте, что появилась `DATABASE_URL` (автоматически)
- [ ] При необходимости добавьте `PORT=5000`

### 3. Перезадеплойте
- [ ] Railway должен автоматически перезадеплоить
- [ ] Или вручную нажмите "Deploy" во вкладке "Deployments"

### 4. Проверьте работу
- [ ] Откройте URL вашего приложения
- [ ] Должна появиться страница авторизации
- [ ] Войдите как админ:

### 5. Безопасность (ВАЖНО!)
- [ ] Смените пароль администратора в `app.py`

## Если всё ещё ошибка 500

### Проверьте логи:
- [ ] Railway Dashboard → Ваш проект → "View Logs"
- [ ] Найдите сообщения об ошибках

### Частые проблемы:
- [ ] Нет DATABASE_URL → Добавьте PostgreSQL
- [ ] Ошибка порта → Не устанавливайте PORT вручную
- [ ] БД не инициализирована → Проверьте логи на "Error during app initialization"
- [ ] SSL ошибка → Код должен добавить ?sslmode=require автоматически
